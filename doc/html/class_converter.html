Opis szczegółowy

Jest to główna klasa abstrakcji konwersji walutowej.

Klasa ta:

  • pełni rolę fabryki, we wzorcu projektowym factory method, dla pobierania danych wejściowych;
  • wybiera algorytm, zgodnie ze wzorcem projektowym strategy, sposobu pobierania kursu dla danych walut;
  • pełni rolę dyrektora, we wzorcu projektowym builder, przy generowaniu danych wyjściowych.
